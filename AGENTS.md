## Проект: VPNExclude

Назначение

Этот репозиторий содержит Windows-утилиту для управления исключениями маршрутизации из WireGuard VPN.

Проект переводится с рабочего PowerShell-скрипта на обычное Windows-приложение на C# WinForms, чтобы пользователи запускали нормальный графический интерфейс, а не скрипт вручную.

Текущая PowerShell-реализация считается эталонным источником бизнес-логики.

C# приложение является новым фронтендом и должно забирать на себя логику постепенно и безопасно.

## Основная идея

Приложение должно управлять исключениями из маршрутизации WireGuard VPN.

Типовая запись исключения содержит поля:
Target
Type
Ips
Gateway
Comment
CreatedAt
UpdatedAt

Для доменов возможно несколько IP-адресов.

Ключевое бизнес-правило из PowerShell-версии:
старые IP и доменные записи нельзя случайно затирать или терять при обновлении.
Новые данные должны объединяться аккуратно.

## Обновление /AGENTS_UPDATE.md после каждой задачи
читать/записывать обновления в файл /AGENTS_UPDATE.md

**Правило:** после КАЖДОЙ завершённой задачи в репозитории добавляем краткую запись в `/AGENTS_UPDATE.md`.  
`AGENTS.md` обновляем ТОЛЬКО если в ходе работы появились новые знания/процедуры/договорённости (плейбук проекта).

### Что именно добавляем в /AGENTS_UPDATE.md
- **Новые инструкции/процедуры:** шаги, команды, флаги, переменные окружения.
- **Неочевидные нюансы:** «грабли», кэш, права на папки, версии пакетов, порядок запуска.
- **Изменения схем/эндпоинтов:** новые поля БД, форматы JSON, коды ошибок, маршруты.
- **Патчи верстки/JS:** конкретные селекторы, CSS-правила, точки инициализации.
- **Чек-листы регресса:** что проверяем, чтобы убедиться, что не сломали соседние места.

### Формат записи в /AGENTS_UPDATE.md (коротко, до 6 строк)
1) `YYYY-MM-DD — <Название задачи/секции>`
2) `Files: <перечень ключевых файлов через запятую>`
3) `- <что сделано 1>`
4) `- <что сделано 2>` (опционально)
5) `PR: <ссылка>` (если есть)
6) `Commit: <хэш>` (если есть)

**Важно:**
- Скриншоты/цитаты/простыни НЕ добавлять.
- Если PR/Commit ещё не создан — строки `PR/Commit` пропустить (не писать `pending`).
- Записи в журнале — **append-only** (старые не переписываем).

### Ротация журнала
- Агент пишет ТОЛЬКО в `/AGENTS_UPDATE.md`.
- При росте `/AGENTS_UPDATE.md` (≈ > 500–800 строк) переносим старые записи в архив:
  - `/AGENTS_UPDATE_ARCHIVE_01.md`, `/AGENTS_UPDATE_ARCHIVE_02.md` и т.д.
- Архивы агент НЕ редактирует.
